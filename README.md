## Overview

This repository contains curated documentation and architectural guidance for CloudOps teams. Documentation is written in Markdown with diagrams created using Mermaid to ensure clarity and accessibility.

## Repository Structure

- **Documentation files** are organized by topic or domain
- **Diagrams and charts** use Mermaid syntax for version control and easy maintenance
- **Code examples** are embedded within relevant documentation sections

## Getting Started

- Review the [Copilot Instructions](.copilot-instructions.md) for guidance on contributing to this repository
- Documentation uses standard Markdown with Mermaid for diagrams
- All contributions should follow the established format and style guidelines

## Contributing

When adding new documentation:
1. Use clear, descriptive headings and sections
2. Include Mermaid diagrams for visual explanations where appropriate
3. Provide examples and use cases
4. Link related documentation
5. Ensure all files are committed to git

## Documentation Standards

- **Markdown**: All documentation files use `.md` extension
- **Diagrams**: Use Mermaid syntax for charts, flowcharts, and architecture diagrams
- **Code Blocks**: Include language specification for syntax highlighting
- **Links**: Use relative links between documentation files
